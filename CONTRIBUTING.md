# Contributing to Euler OMR

First off, thank you for considering contributing to Euler OMR! It's people like you that make Euler OMR such a great tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for Euler OMR. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related bugs.

Before creating bug reports, please check if there is an existing issue that reports the same problem. When you are creating a bug report, please include as many details as possible. Fill out the bug report template, as this helps us resolve issues faster.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion for Euler OMR, including completely new features and minor improvements to existing functionality. Following these guidelines helps maintainers and the community understand your suggestion and find related suggestions.

Before creating enhancement suggestions, please check if there is an already existing issue that describes the same feature or a similar one.

### Pull Requests
- Fill in the Pull Request template.
- Ensure the code follows the existing style (PEP 8 for Python).
- If you're adding a new feature, please update the documentation (README) accordingly.
- Your PR should ideally be focused on a single change.

## Styleguides

### Python Styleguide
We follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

## Any Questions?
Don't hesitate to open an issue or reach out to the project maintainers!
